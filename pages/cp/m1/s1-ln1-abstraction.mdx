# Abstraction
<br />

**Abstraction** is one of the fundamental core components of Computer Science,
together with Decomposition, Pattern Recognition, and Algorithms. The essence
of Abstraction involves hiding all the features that we donâ€™t need in order to
focus on those that are more important. From this, we create a representation
of what we are trying to solve and develop algorithms.

<br />

Abstraction can be applied to any aspect of our life. For example, to print a document, users can just simply press the button â€œPrintâ€ without understanding how the machine applies ink onto the paper. The button â€œPrintâ€ is a representation of the whole printing process. By using abstraction, the engineer hides all the unnecessary detail about the printing process from the users.

<br />
Abstraction can be applied in e-commerce to help a business operate in a more systematic
and efficient way. All the information about inventories and customers, for example,
can be recorded and organized in tables. In these tables, only important features
are highlighted while all the other irrelevant information is filtered out). For
example, in fig1â€™s, the PEOPLE table only records important fields such as Address,
Email, etc., and ignores other irrelevant ones like â€œHair Colorâ€, and â€œPetsâ€.
<br />
From these tables, we can develop a system called ERD (Entity Relationship Diagram).
Based on this, we can further understand the relationship within our data. For example,
the links between tables show us which customer makes which order of which product,
and their review about the product. ERD is an example of how abstraction can help
management understand their data before diving deeper into details.
