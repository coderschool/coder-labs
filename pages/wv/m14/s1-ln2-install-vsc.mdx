# Config Local Development Environment

<br />
<br />

## What will you need

- Chrome as the browser. You can install it from <a href="https://www.google.com/chrome/" target="_blank">here</a>. Chrome includes a set of developer tools that will come in handy down the road.

- Visual Studio Code (VS Code) as the code editor. Follow this <a href="https://code.visualstudio.com/docs/setup/setup-overview" target="_blank">setup instruction</a>. Pick the guide that matches your operating system.

  Open VS Code and access the Command Palette (Ctrl/Cmd + Shift + P) and start typing `shell command` and select option `Shell Command: Install 'code' command in PATH`.

  ![Code command](https://docs.coderschool.vn/assets/web-virgil/vscode_1.png)

- Git and GitHub for version control and repository hosting service. Getting started <a href="https://git-scm.com/book/en/v2/Getting-Started-Installing-Git" target="_blank">here</a>.

- NodeJS as the Javascript run-time environment. Go to the <a href="http://nodejs.org/" target="_blank">Node home page</a>. Click the big green button that has a version number followed by "LTS (Recommended for Most Users)."

- Using the Terminal (also called a console or command prompt). If you're not friends with your terminal, I highly recommend you familiarize yourself with your terminal of choice. If you're on macOS or Linux, you already have it. For Windows users, Git helpfully includes a "Git Bash" shell, which provides a Unix-like terminal experience.

> Note: here is another instruction for Windows user: [[Setup Environment for Windows|Environment Setup Window]]

<br />

## Testing your environment

Open a Terminal:

- Test that `nodejs` is installed: run the "version" command

```
$ node -v
v12.19.0

```

- The Nodejs package manager NPM should also have been installed, and can be tested in the same way:

```
$ npm -v
6.14.6

```

- The version command in `git` is slightly different:

```
$ git --version
git version 2.24.3

```

- Run this command to open the current directory in Visual Studio Code

```
$ code .

```

## VS Code Extensions

Extensions are an easy way to extend the out-of-the-box functionality of Visual Studio Code. You can browse and install VS Code extensions from the Extensions view in the Side Bar. To open that view, just click on the Extension Icon in the Activity Bar on the left. The following view will open:

![VSCode Extension](https://docs.coderschool.vn/assets/web-virgil/vscode_2.png)

In the extensions view, you can search for extensions. The list of results is presented in the Side Bar area. You can select an extension from the list; the extensions details page is shown on the right.

This is the list of extensions that will be useful:

- Live Server: Launch a local development Server with a live reload feature for static & dynamic pages.

![Live Server](https://docs.coderschool.vn/assets/web-virgil/vscodeex_live_server.png)

- Prettier: to prettify your code.

![Prettier](https://docs.coderschool.vn/assets/web-virgil/vscodeex_prettier.png)

- ES7 React/Redux/GraphQL/React-Native snippets: This extension provides you JavaScript and React/Redux snippets in ES7 with Babel plugin features for VS Code.

![ES7 snippets](https://docs.coderschool.vn/assets/web-virgil/vscodeex_es7_react.png)

- ESLint: Integrates ESLint JavaScript into VS Code.

![ESLint](https://docs.coderschool.vn/assets/web-virgil/vscodeex_eslint.png)

- Bracket Pair Colorizer 2: A customizable extension for colorizing matching brackets.

![Bracket Pair](https://docs.coderschool.vn/assets/web-virgil/vscodeex_bracket_pair.png)

- Auto Close Tag: Automatically add HTML/XML close tag

![Auto Close Tag](https://docs.coderschool.vn/assets/web-virgil/vscodeex_auto_close_tag.png)

- Auto Rename Tag: Automatically rename pair HTML/XML tag

![Auto Rename Tag](https://docs.coderschool.vn/assets/web-virgil/vscodeex_auto_rename_tag.png)

- Color Highlight: Highlight web colors in your editor.

![Color Highlight](https://docs.coderschool.vn/assets/web-virgil/vscodeex_color_highlight.png)

- DotENV: Support for dotenv file syntax.

![DotENV](https://docs.coderschool.vn/assets/web-virgil/vscodeex_dotenv.png)

Last but not least, we need to config a few settings for the extensions. The shortest way is:

- `Ctrl/Cmd + Shift + P`, and find `Open Settings(JSON)`

![Open Settings](https://docs.coderschool.vn/assets/web-virgil/vscode_open_settings.png)

- It will open the `settings.json` file, which you can customize to VSCode's default settings. Paste this JSON into it:

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.formatOnPaste": false,
  "[json]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "json.format.enable": false,
  "window.zoomLevel": 1,
  "editor.fontSize": 14,
  "editor.tabSize": 2,
  "terminal.integrated.fontSize": 14,
  "emmet.includeLanguages": {
    "javascript": "javascriptreact"
  },
  "javascript.updateImportsOnFileMove.enabled": "always",
  "editor.tabCompletion": "on",
}
```
