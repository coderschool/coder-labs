# Flexbox

<br />
<br />

## Introduction

<br />

### Why use Flexbox?

Flexbox is here to provide a consistent layout on different screen sizes as your users' screen would varied from mobile to desktop, and ect.

That means it allows items to stretch or shrink inside their containers, preventing wasted space and overflow. It's a real plus for making layouts fit a variety of viewport sizes.

Other advantages:

> ‚úÖ Make all neighboring items the same height.<br/>
> ‚úÖ Easy horizontal and vertical centering.<br/>
> ‚úÖ The ability to change the order in which items display, independent of the source.<br/>

### Getting to know Flexbox

Flexbox begins with the `display` property.
By applying `display: flex` to an element, you turns it into a **flex container**, and its direct children into **flex items**.

By default, flex items **align side by side, left to right, all in one row**.
The flex container fills the available width like a block element, but the flex items may not necessarily fill the width of their flex container. The flex items are all the same height, determined naturally by their contents.

Because flexbox layout is defined in terms of the main axis and cross axis, therefore, we use the terms `start` and `end` in reference to the axes. There is no `left` and `right` or `top` and `bottom` in flexbox.

### 1Ô∏è‚É£ `flex: flex-grow flex-shrink flex-basis`

> - This is the short hand for `flex-grow`, `flex-shrink` and `flex-basis` combined. It is recommended that you use this shorthand property rather than set the individual properties.
> - For the `flex-grow` and `flex-shrink` properties, the values 1 and 0 work like on/off switches, where 1 "turns on" or allows an item to grow or shrink, and 0 prevents it.
> - The `flex-basis` property sets the starting dimensions, either to a specific size or `auto` - a size based on the contents.

<CH.Scrollycoding preset="https://codesandbox.io/s/flexbox-2uownl">
